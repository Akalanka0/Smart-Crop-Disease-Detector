## 🎯 What Is This?

This project is a **ready-to-use template** for building a crop disease detection system.  
It is **not** limited to any single crop — just bring your own image dataset and run.

**Included demo:** Tomato disease classifier (10 classes from PlantVillage dataset).

### What you get
| Feature | Details |
|---|---|
| 🧠 **Model** | Fine-tuned ResNet-50 (ImageNet pretrained) |
| 🔁 **Training pipeline** | Advanced Augmentation (Blur/Perspective) · cosine LR schedule |
| 🌐 **Web UI** | Dark-themed drag-and-drop interface |
| 🔌 **REST API** | Flask backend with `/predict`, `/classes`, `/health` endpoints |
| 🛡️ **Plant Guard** | CLIP-based verification to reject non-plant images |

---

## 🛡️ Plant Guard (Image Verification)

This template includes a **Plant Guard** system to prevent misclassifications. If a user uploads a photo of a chair, car, or person, the system uses **OpenAI's CLIP** to detect that it isn't a plant and rejects the input before it reaches the disease classifier.

**To enable this feature:**
```bash
pip install git+https://github.com/openai/CLIP.git
```
*If CLIP is not installed, the system will fall back to using stricter confidence thresholds.*

## 🚀 Quick Start (Use With Your Own Dataset)

### 1 · Clone the repo

```bash
git clone https://github.com/YOUR_USERNAME/crop-disease-detector.git
cd crop-disease-detector
```

### 2 · Install dependencies

```bash
python -m venv venv
# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate

pip install -r requirements.txt
```

### 3 · Add your dataset

Organise images in folders — **one folder per class**:

```text
dataset/
├── Rice_Blast/
│   ├── img001.jpg
│   └── img002.jpg
├── Rice_Healthy/
│   └── img004.jpg
└── Background/
    └── random_chair.jpg   <-- (Optional: Add non-leaf images to prevent false positives)
```

> The folder names become your class labels automatically.  
> Any crop works: Rice, Maize, Wheat, Grape, Potato, etc.  
> **Pro Tip:** Add a `Background` folder with 50+ random images (hands, soil, tables) to force the model to explicitly learn and reject non-leaves!


### 4 · Train the model

```bash
python train.py
```

This produces:
- `model_best.pth` — best checkpoint
- `class_names.json` — class list used by predict.py and the backend
- `training_history.json` — epoch-by-epoch metrics
- `training_curves.png` — loss & accuracy plots

### 5 · Start the web app

```bash
python backend/app.py
```

Open **http://localhost:5000** in your browser. That's it. ✅

## 📡 API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET`  | `/`         | Serves the web UI |
| `POST` | `/predict`  | Predict disease from uploaded image |
| `GET`  | `/classes`  | List all trained class names |
| `GET`  | `/health`   | Server status, device, class count |

**POST `/predict`** — send as `multipart/form-data` with key `image`:

```bash
curl -X POST http://localhost:5000/predict -F "image=@leaf.jpg"
```

Response:
```json
{
  "status": "ok",
  "class": "Rice_Blast",
  "label": "Rice Blast",
  "confidence": 94.72,
  "entropy": 0.312,
  "top5": [["Rice_Blast", 94.72], ["Rice_Brown_Spot", 3.11], ...]
}
```

## ⚙️ Configuration

All training settings are in the `CFG` dict at the top of `train.py`:

```python
CFG = {
    "dataset_dir":  "dataset",   # ← Point to your dataset folder
    "img_size":     224,
    "batch_size":   32,
    "epochs":       30,
    "lr":           1e-3,
    "weight_decay": 1e-4,
    "patience":     6,           # Early stopping patience
    "val_split":    0.2,
    "num_workers":  0,           # Keep 0 on Windows
    "save_path":    "model_best.pth",
}
```

## 🧠 Model Architecture

```
Input Image (224×224)
        ↓
ResNet-18 Backbone  (ImageNet pretrained)
  · Layer 1 & 2     → frozen
  · Layer 3 & 4     → fine-tuned
        ↓
Dropout(0.3)  →  Linear(512 → N classes)
        ↓
Softmax  →  Predicted Class + Confidence
```

**Training strategy:**
- Cosine Annealing LR scheduler
- Label smoothing (ε = 0.1) for better generalisation
- AdamW optimizer with L2 weight decay
- Early stopping on validation accuracy

## 🔒 CLIP Input Guard (Optional)

`predict.py` includes an optional CLIP-based gate that rejects non-plant images before they reach the classifier.

Enable it:
```bash
pip install git+https://github.com/openai/CLIP.git
```

Set `USE_CLIP_GUARD = True` in `predict.py` (default: `True`).  
The web backend does **not** use CLIP (for speed).
